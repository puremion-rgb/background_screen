# 📑 크롬 확장프로그램 개발 기획서

## ① 서비스 정의

- **서비스명:** Shopping Shot (쇼핑몰 이미지 스크랩 및 가격 비교 도구)
- **타깃 사용자:** 여러 쇼핑몰(쿠팡, 네이버쇼핑, 무신사 등)을 돌아다니며 최저가를 비교하고 신중하게 소비하는 '스마트 쇼핑족'
- **기획 배경:** 사고 싶은 물건이 생겼을 때 여러 사이트의 탭을 오가며 가격을 비교하는 것은 번거롭습니다. 마음에 드는 상품 이미지 위에서 우클릭 한 번으로 이미지를 박스에 담고, 가격을 기록하여 한눈에 보며 비교할 수 있는 나만의 '쇼핑 장바구니'를 제공하고자 합니다.

---

## ② 기능 정의

1. **상품 이미지 스크랩 (우클릭 메뉴):** 쇼핑몰의 상품 이미지 위에서 마우스 우클릭을 하면 해당 이미지의 URL과 현재 페이지 주소를 추출합니다.
2. **가격 및 상품명 입력 (Prompt 팝업):** 이미지를 스크랩할 때, 화면에 입력창(Prompt)을 띄워 사용자가 직접 현재 가격과 상품명을 간편하게 입력할 수 있도록 합니다.
3. **쇼핑 리스트 시각화 (UI 팝업):** 브라우저 우측 상단의 익스텐션 아이콘을 누르면, 스크랩한 상품 이미지들이 카드 형태(Grid)로 나열되며 기록한 가격과 함께 표시됩니다.
4. **가격 비교 및 링크 이동:** 저장된 상품 카드를 클릭하면 원래 구매하려고 했던 쇼핑몰 상세 페이지로 바로 이동합니다.
5. **데이터 관리:** 이미 구매했거나 리스트에서 제외하고 싶은 상품은 '삭제' 버튼을 통해 지울 수 있습니다.

---

## ③ UX 설계

1. **상품 발견:** 사용자가 쇼핑몰에서 마음에 드는 운동화 이미지를 발견합니다.
2. **우클릭 스크랩:** 운동화 이미지 위에서 우클릭한 후 `[Shopping Shot에 상품 추가]` 메뉴를 클릭합니다.
3. **정보 입력:** 현재 보고 있는 웹페이지 화면에 입력창이 뜨면, 상품명과 가격을 간단히 입력하고 확인을 누릅니다. (취소 시 저장하지 않음)
4. **비교 및 결정:** 브라우저 우측 상단 익스텐션 아이콘을 열어 팝업창(UI)을 켭니다. 다른 쇼핑몰에서 스크랩한 동일 운동화 카드들과 비주얼 및 가격을 직관적으로 비교한 후, 가장 저렴한 카드를 클릭해 해당 쇼핑몰로 이동하여 결제합니다.

## ④ 기술 명세

- **Manifest Version:** MV3 (Manifest Version 3)
- **핵심 권한 (Permissions):** `contextMenus`, `storage`, `scripting`, `activeTab`
- **주요 기능 구현 방식:**
  - **우클릭 메뉴 구성:** `chrome.contextMenus.create`의 `contexts: ["image"]` 속성을 활용하여, 사용자가 **'이미지 위에서 우클릭했을 때만'** 메뉴가 나타나도록 정밀하게 제어합니다.
  - **데이터 수집:** 우클릭 시 `info.srcUrl`(이미지 주소)과 `info.pageUrl`(쇼핑몰 상품 주소)을 동시에 확보합니다.
  - **데이터 저장:** `chrome.storage.local`을 사용하여 `[{ id, title, price, imgUrl, pageUrl, date }]` 구조의 JSON 배열로 브라우저에 영구 저장합니다.
  - **UI 형태:** `popup.html` 내부에 그리드(Grid) 레이아웃을 적용하여 쇼핑몰처럼 깔끔한 상품 카드 리스트 UI를 구현합니다.
  - 일부 쇼핑몰의 우클릭 차단 스크립트를 무력화하기 위해 content.js에서 이벤트 전파 차단(stopPropagation) 기술을 적용하여 호환성을 극대화함
